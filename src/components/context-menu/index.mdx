# React Context Menu

The `ContextMenu` component provides a popup menu that appears when a user interacts with an element through right-click or a long press action. It offers a convenient way to present contextual options and commands relevant to the selected element or area.

## Custom Icons

Beyond simple text menu items, the `ContextMenu` supports rich content through component composition within `MenuItem` and the [`itemTemplate`](https://react-api.syncfusion.com/context-menu/overview#itemtemplate) prop. Using component composition allows for custom content in each menu item, while the [`itemTemplate`](https://react-api.syncfusion.com/context-menu/overview#itemtemplate) prop provides a consistent template for all items. Both approaches enable advanced customization with icons, layout formatting, and custom styling to create visually distinctive menu interactions.

## Animation Effects

The [`animation`](https://react-api.syncfusion.com/context-menu/overview#animation) property enhances user experience by providing visual feedback when menus appear. ## Scrollable Menu

For data-heavy menus, the [`items`](https://react-api.syncfusion.com/context-menu/overview#items) property enables programmatic menu creation using the [`MenuItemProps`](https://react-api.syncfusion.com/context-menu/menuitemprops#props) interface as an alternative to declarative definition. The `className` prop can be used with CSS to create scrollable menus when content exceeds view limits, with a default max height of 500px that can be customized (shown here with a 200px constraint).

### WAI-ARIA attributes

The `ContextMenu` component followed the [WAI-ARIA](https://www.w3.org/WAI/ARIA/apg/patterns/menubar/) patterns to meet the accessibility. The following ARIA attributes are used in the `ContextMenu` component:

| Attributes | Purpose |
| --- | --- |
| <mark>role</mark> | Indicates Context menu component popup as `menu`, and the popup items as `menuitem`. |
| <mark>aria-haspopup</mark> | Indicates the availability and type of interactive popup element. |
| <mark>aria-expanded</mark> | Indicates whether the subtree can be expanded or collapsed, as well as indicates whether its current state is expanded or collapsed. |
| <mark>aria-label</mark> | Indicates the menu item text. |
| <mark>aria-disabled</mark> | Indicates element is perceivable but disabled. |

### Keyboard interaction

The `ContextMenu` component follows the [keyboard interaction](https://www.w3.org/WAI/ARIA/apg/patterns/menubar/#keyboardinteraction) guidelines, making it accessible for people who use assistive technologies (AT) and those who rely on keyboard navigation. The following keyboard shortcuts are supported:

| **Windows** | **Mac** | **To do this** |
| --- | --- | --- |
| <kbd>Esc</kbd> | <kbd>Esc</kbd> | Closes the opened sub menu. |
| <kbd>Enter , Space</kbd> | <kbd>Enter , Space</kbd>| Selects the focused item.|
| <kbd>↑</kbd> | <kbd>↑</kbd>| Navigates up or to the previous menu item. |
| <kbd>↓</kbd>| <kbd>↓</kbd>  | Navigates down or to the next menu item. |
| <kbd>←</kbd>| <kbd>←</kbd>  | Closes the active submenu and returns to the parent menu |
| <kbd>→</kbd>|<kbd>→</kbd> | Navigates and open the next sub menu. |
| <kbd>Home</kbd> | <kbd> Fn + ←</kbd> | Moves focus to first item in the menu. |
| <kbd>End</kbd> | <kbd>Fn + → </kbd> | Moves focus to last item in the menu. |
| <kbd>Character</kbd> | <kbd>Character</kbd> | Moves focus to next item in the menu having a name that starts with the typed character.<br></br> If none of the items have a name starting with the typed character, focus does not move. |
